# L1 Requirements - CarSearch

High-level requirements for the CarSearch application.

## L1-REQ-001: Vehicle Inventory Search

The system shall search used vehicle inventory across multiple dealership websites and return aggregated results matching user-specified criteria.

## L1-REQ-002: Search Parameters

The system shall accept search criteria including vehicle make, model, postal code, and optional filters for color, model year range, and search timeout.

## L1-REQ-003: Plugin-Based Provider Architecture

The system shall support a plugin architecture where each dealership is an independent, self-contained provider that can be added, removed, or enabled/disabled without modifying core application code.

## L1-REQ-004: Browser Automation

The system shall interact with dealership websites using headless browser automation via playwright-cli to navigate pages, apply filters, and extract inventory data.

## L1-REQ-005: ARIA Snapshot Parsing

The system shall extract structured vehicle listing data from ARIA accessibility tree snapshots of dealership web pages using pattern-based parsing.

## L1-REQ-006: Report Generation

The system shall produce a consolidated markdown report containing all search results, organized by provider, with summary statistics.

## L1-REQ-007: Configuration Management

The system shall be configurable via a JSON settings file, allowing per-provider enable/disable control, base URLs, and playwright-cli settings without code changes.

## L1-REQ-008: Fault Tolerance

The system shall continue searching remaining providers when an individual provider encounters an error, ensuring partial results are still returned.

## L1-REQ-009: Command-Line Interface

The system shall be operable as a CLI application with clearly defined arguments and options.

## L1-REQ-010: Sequential Provider Execution

The system shall execute provider searches sequentially to avoid shared browser resource conflicts.
