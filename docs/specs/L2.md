# L2 Requirements - CarSearch

Detailed requirements with traceability to L1 and acceptance criteria.

---

## L2-REQ-001: Multi-Provider Search Orchestration

**Traces to:** L1-REQ-001, L1-REQ-010

The ProviderOrchestrator shall collect all registered `ICarSearchProvider` instances, filter to enabled providers, and execute each provider's `SearchAsync` method sequentially, aggregating results into a list of `ProviderSearchResult`.

### Acceptance Criteria

- AC-001.1: Only providers where `IsEnabled` returns `true` are executed.
- AC-001.2: Providers execute one at a time, never concurrently.
- AC-001.3: The orchestrator returns a `List<ProviderSearchResult>` containing one entry per executed provider.
- AC-001.4: Each result includes provider name, display name, listings, total count, success status, and duration.

---

## L2-REQ-002: Required Search Parameters

**Traces to:** L1-REQ-002, L1-REQ-009

The CLI shall require `--make`, `--model`, and `--postal-code` arguments. The application shall not execute a search if any required parameter is missing.

### Acceptance Criteria

- AC-002.1: The application exits with an error message when `--make` is omitted.
- AC-002.2: The application exits with an error message when `--model` is omitted.
- AC-002.3: The application exits with an error message when `--postal-code` is omitted.
- AC-002.4: All three values are passed to providers via the `SearchParameters` object.

---

## L2-REQ-003: Optional Search Filters

**Traces to:** L1-REQ-002, L1-REQ-009

The CLI shall accept optional `--color`, `--year-from`, `--year-to`, `--timeout`, and `--output` arguments that refine search behavior or control output.

### Acceptance Criteria

- AC-003.1: When `--color` is provided, providers that support color filtering apply it to their search.
- AC-003.2: When `--year-from` is provided, `SearchParameters.YearFrom` is set and providers filter listings accordingly.
- AC-003.3: When `--year-to` is provided, `SearchParameters.YearTo` is set and providers filter listings accordingly.
- AC-003.4: When `--timeout` is provided, the value (in milliseconds) is used as the playwright-cli command timeout. Default is 15000ms.
- AC-003.5: When `--output` is provided, the report is written to the specified file path.

---

## L2-REQ-004: Provider Interface Contract

**Traces to:** L1-REQ-003

Each provider shall implement the `ICarSearchProvider` interface exposing `Name`, `DisplayName`, `IsEnabled`, and `SearchAsync(SearchParameters, CancellationToken)`.

### Acceptance Criteria

- AC-004.1: `Name` returns a unique machine-friendly identifier string.
- AC-004.2: `DisplayName` returns a human-readable dealership name.
- AC-004.3: `IsEnabled` reflects the `Enabled` flag from the provider's configuration section.
- AC-004.4: `SearchAsync` returns a `ProviderSearchResult` containing zero or more `VehicleListing` objects.
- AC-004.5: `SearchAsync` respects the provided `CancellationToken`.

---

## L2-REQ-005: Provider Registration and Configuration

**Traces to:** L1-REQ-003, L1-REQ-007

Each provider shall be registered in the DI container as a transient `ICarSearchProvider` with a corresponding named `ProviderOptions` bound to the `Providers:{Name}` configuration section.

### Acceptance Criteria

- AC-005.1: Each provider's `ProviderOptions` is loaded from `appsettings.json` under `Providers:{ProviderName}`.
- AC-005.2: `ProviderOptions` includes `Enabled` (bool), `BaseUrl` (string), and `Settings` (dictionary).
- AC-005.3: A provider can be disabled by setting `Enabled: false` in `appsettings.json` without removing code.
- AC-005.4: The provider and its snapshot parser are both registered as transient services.

---

## L2-REQ-006: Playwright-CLI Browser Interaction

**Traces to:** L1-REQ-004

The `PlaywrightCliService` shall wrap playwright-cli commands to open URLs, take snapshots, click elements, fill inputs, select dropdowns, and close browser sessions.

### Acceptance Criteria

- AC-006.1: `OpenAsync(url)` launches a headless browser and navigates to the given URL.
- AC-006.2: `SnapshotAsync()` captures the ARIA accessibility tree and returns the YAML content as a string.
- AC-006.3: `ClickAsync(refId)` clicks the element identified by the given ARIA snapshot ref ID.
- AC-006.4: `FillAsync(refId, text)` fills a form input identified by ref ID with the given text.
- AC-006.5: `SelectAsync(refId, value)` selects a dropdown option by value.
- AC-006.6: `CloseAsync()` terminates the browser session and releases resources.
- AC-006.7: Commands that exceed the configured timeout throw a `TimeoutException`.
- AC-006.8: Commands that return a non-zero exit code throw an `InvalidOperationException`.

---

## L2-REQ-007: Snapshot Retry Logic

**Traces to:** L1-REQ-004, L1-REQ-008

The `PlaywrightCliService` shall support retrying ARIA snapshot capture on failure, with configurable retry count and delay.

### Acceptance Criteria

- AC-007.1: `SnapshotWithRetryAsync()` retries up to `RetryCount` times (default 3) on failure.
- AC-007.2: A delay of `RetryDelayMs` (default 2000ms) is applied between retry attempts.
- AC-007.3: If all retries are exhausted, the last exception is propagated.
- AC-007.4: Retry count and delay are configurable via `PlaywrightCliOptions`.

---

## L2-REQ-008: ARIA Snapshot Parsing - Vehicle Listings

**Traces to:** L1-REQ-005

Each provider's `SnapshotParser` shall extract structured `VehicleListing` data from ARIA snapshot YAML using regex patterns specific to the dealership platform.

### Acceptance Criteria

- AC-008.1: The parser extracts vehicle title, year, price, mileage, transmission, and listing URL from the ARIA snapshot.
- AC-008.2: URLs are resolved to absolute URLs using the provider's `BaseUrl`.
- AC-008.3: Only listings with year >= 2000 are included in results.
- AC-008.4: The parser handles missing or malformed fields gracefully, returning partial data rather than throwing.

---

## L2-REQ-009: ARIA Snapshot Parsing - Filter Discovery

**Traces to:** L1-REQ-005

Each provider's `SnapshotParser` shall locate interactive filter elements (listitems, dropdowns, buttons) by label text and return their ref IDs for click/select operations.

### Acceptance Criteria

- AC-009.1: `FindListItemRef(yaml, label)` returns the ref ID of a listitem matching the given label, or `null` if not found.
- AC-009.2: Filter discovery uses case-appropriate matching for the dealership platform.
- AC-009.3: The parser can locate make, model, and color filter elements when present in the snapshot.

---

## L2-REQ-010: ARIA Snapshot Parsing - Metadata Extraction

**Traces to:** L1-REQ-005

Each provider's `SnapshotParser` shall extract metadata such as result count and city/location from the ARIA snapshot.

### Acceptance Criteria

- AC-010.1: `ParseResultCount(yaml)` returns the integer count of matching vehicles, or 0 if not found.
- AC-010.2: `ParseCity(yaml)` returns the city name from page headings, or `null` if not found.

---

## L2-REQ-011: Markdown Report Structure

**Traces to:** L1-REQ-006

The `MarkdownReportGenerator` shall produce a markdown file containing a header, search metadata, provider summary table, per-provider listing tables, and an aggregated summary.

### Acceptance Criteria

- AC-011.1: The report header includes make, model, location, and year range (if specified).
- AC-011.2: Search metadata includes search date, active sources, applied filters, and total result count.
- AC-011.3: A provider summary table shows each provider's name, result count, duration, and status (success/failure).
- AC-011.4: Each provider with results has a section containing a numbered list or table of listings with title, URL, price, mileage, transmission, and fuel type.
- AC-011.5: An aggregated summary section includes new/used breakdown, price ranges, price bracket distribution, and year distribution.

---

## L2-REQ-012: Report Output Path

**Traces to:** L1-REQ-006, L1-REQ-009

The system shall write the report to a user-specified path or auto-generate a descriptive filename based on search parameters.

### Acceptance Criteria

- AC-012.1: When `--output` is provided, the report is written to that exact path.
- AC-012.2: When `--output` is omitted, the filename follows the pattern `{color}-{make}-{model}-{city}.md`.
- AC-012.3: The application prints the output file path to the console upon completion.

---

## L2-REQ-013: Provider Error Isolation

**Traces to:** L1-REQ-008

The orchestrator shall catch exceptions thrown by individual providers and record the failure without halting the overall search.

### Acceptance Criteria

- AC-013.1: When a provider throws an exception, its `ProviderSearchResult.Success` is set to `false`.
- AC-013.2: The exception message is stored in `ProviderSearchResult.ErrorMessage`.
- AC-013.3: Remaining providers continue to execute after a provider failure.
- AC-013.4: The final report includes failed providers with their error status in the summary table.

---

## L2-REQ-014: Playwright-CLI Configuration

**Traces to:** L1-REQ-007

The `PlaywrightCliOptions` shall be loaded from the `PlaywrightCli` section of `appsettings.json` and provide the CLI command path, default timeout, retry count, and retry delay.

### Acceptance Criteria

- AC-014.1: `Command` specifies the full filesystem path to the playwright-cli executable.
- AC-014.2: `DefaultTimeoutMs` sets the per-command timeout (default 15000ms), overridable by the `--timeout` CLI argument.
- AC-014.3: `RetryCount` sets the maximum snapshot retry attempts (default 3).
- AC-014.4: `RetryDelayMs` sets the delay between retries (default 2000ms).

---

## L2-REQ-015: Vehicle Listing Data Model

**Traces to:** L1-REQ-001, L1-REQ-005

Each parsed vehicle listing shall be represented as a `VehicleListing` containing title, price, mileage, transmission, fuel type, URL, dealer, location, year, new/used status, and source provider.

### Acceptance Criteria

- AC-015.1: `Title` contains the full vehicle description (e.g., "2024 Land Rover Range Rover").
- AC-015.2: `Price` contains the listing price as a string.
- AC-015.3: `Year` contains the integer model year extracted from the listing.
- AC-015.4: `Url` contains an absolute URL to the listing page.
- AC-015.5: `Source` identifies the provider that produced the listing.
- AC-015.6: `IsNew` distinguishes new from used/pre-owned vehicles.

---

## L2-REQ-016: D2C Media Platform Support

**Traces to:** L1-REQ-003, L1-REQ-005

Providers for D2C Media dealership sites shall navigate `/used/search.html`, interact with sidebar listitem filters, and parse listings from the `link "YEAR Model in Location"` pattern.

### Acceptance Criteria

- AC-016.1: The provider opens `{BaseUrl}/used/search.html`.
- AC-016.2: Make filter is applied by finding and clicking the matching listitem.
- AC-016.3: Model filter is applied by finding and clicking the matching listitem.
- AC-016.4: Color filter is applied when `--color` is specified and the color listitem exists.
- AC-016.5: Listings are extracted from `link "YEAR Model in Location"` patterns with associated price and spec text.

---

## L2-REQ-017: LeadBox/WordPress Platform Support

**Traces to:** L1-REQ-003, L1-REQ-005

Providers for LeadBox/WordPress dealership sites shall navigate inventory pages with dropdown filters and parse listings from multi-element patterns (separate links for year, model, and generic elements for price/specs).

### Acceptance Criteria

- AC-017.1: The provider navigates to the configured inventory URL.
- AC-017.2: Dropdown filters are located and interacted with via ref IDs.
- AC-017.3: Year is extracted from `link "New/Used YEAR"` or `link "YEAR"` elements.
- AC-017.4: Model is extracted from a separate link element.
- AC-017.5: Price and spec details (colour, engine, drivetrain) are extracted from generic elements.

---

## L2-REQ-018: Dependency Injection Setup

**Traces to:** L1-REQ-003, L1-REQ-007

The application shall use `Microsoft.Extensions.DependencyInjection` to register all providers, parsers, services, and configuration, enabling loosely coupled composition.

### Acceptance Criteria

- AC-018.1: All `ICarSearchProvider` implementations are registered as transient services.
- AC-018.2: All `SnapshotParser` classes are registered as transient services.
- AC-018.3: `ProviderOrchestrator` is registered as a transient service.
- AC-018.4: `MarkdownReportGenerator` is registered as a singleton service.
- AC-018.5: Named `ProviderOptions` are bound from configuration for each provider.
- AC-018.6: Logging is configured with console output at Information level minimum.
